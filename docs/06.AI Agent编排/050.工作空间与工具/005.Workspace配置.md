## 目录结构

每个 conversation 都会在 `liteflow.agent.workspace.root` 下获得一个独立子目录：

```
/var/lib/liteflow/agent-workspaces/
├── chat-user-1-conv-1/     # conversation 的 workspace
├── chat-user-1-conv-2/     # 另一个 conversation
└── .agent-session/         # LOCAL_FILE 模式的记忆文件
```

`<conversationId>/` 是同一段 conversation 中多个 Agent 共享的读写目录。

## 配置项

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `liteflow.agent.workspace.root` | 无（必填） | workspace 根目录 |
| `liteflow.agent.workspace.auto-create` | `true` | 根目录不存在时是否自动创建 |
| `liteflow.agent.workspace.cleanup-on-session-expire` | `true` | Session 过期时是否删除对应 workspace |
| `liteflow.agent.workspace.cleanup-on-jvm-shutdown` | `false` | JVM 关闭时是否删除 workspace |
| `liteflow.agent.workspace.max-file-bytes` | `10485760` | `read_file` 单次最多读取的字节数 |
| `liteflow.agent.workspace.max-list-size` | `1000` | `list_files` 单次最多返回的条目数 |

::: tip 文件长期保留
如果 Agent 生成的文件需要长期保留（报告、草稿、审计材料），应把 `cleanup-on-session-expire` 设为 `false`，并自行做归档。
:::
