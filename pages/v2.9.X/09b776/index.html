📋Nacos配置源

LiteFlow从v2.9.0开始，原生支持了Nacos的规则配置源。

# 依赖

如果使用Nacos作为规则配置源，你需要添加以下额外插件依赖：

<dependency>
    <groupId>com.yomahub</groupId>
    <artifactId>liteflow-rule-nacos</artifactId>
    <version>2.9.7</version>
</dependency>

# 配置

依赖了插件包之后，你无需再配置liteflow.ruleSource路径。

只需要配置插件的额外参数即可：

    liteflow:
  rule-source-ext-data-map:
    serverAddr: 127.0.0.1:8848
    dataId: demo_rule
    group: DEFAULT_GROUP
    namespace: your namespace id
    username: nacos
    password: nacos

    liteflow.rule-source-ext-data={\
    "serverAddr":"127.0.0.1:8848",\
    "dataId":"demo_rule",\
    "group":"DEFAULT_GROUP",\
    "namespace":"your namespace id",\
    "username":"nacos",\
    "password":"nacos"\
}

    // Make sure to add code blocks to your code group

    # 配置说明

    配置项 说明
    serverAddr nacos的连接串
    dataId 整个规则+脚本的xml存储数据节点id
    group nacos的group
    namespace nacos的namespace
    username nacos的用户
    password nacos的密码

    # 存储数据说明

    警告

    需要说明的是，使用Nacos配置源的时候，Nacos里存的只能是xml形式的配置。

    比如：

    <?xml version="1.0" encoding="UTF-8"?>
<flow>
  	<nodes>
      <node id="s1" type="script">
        你的脚本代码
      </node>
    </nodes>
  
    <chain name="chain1">
      THEN(a, b, c);
    </chain>
</flow>

    # 自动刷新

    使用了此Nacos配置源插件，凡是Nacos节点里面的规则改动，会自动推送到业务系统，进行实时的平滑热刷新。你无需做任何事情。

    # 小例子

    为了让大家能简单上手Nacos规则文件的配置和运行，这里有一个小demo，大家可以拉到本地来运行，需要你替换Nacos的配置信息。

    运行项目前，先读项目里的readme.txt文件。

    https://github.com/bryan31/liteflow-ext-rule-demo
